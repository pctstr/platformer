# Changelog


## [1.0.117] — 2026-06-27

### Исправлено
- **Выход влево со Temple:** `world.setBounds(-25, 0, 345, 240)` — теперь переход в Jungle срабатывает (раньше левая граница -25 не давала уйти за -20).
- **Правая стена на Temple:** `setCollideWorldBounds(true)` + bounds.right=320 создают стену на краю экрана (нет следующего уровня).
- **Переходы с траекторией:** при переходе между экранами сохраняются `velX/velY` — полёт продолжается на новом уровне, игрок визуально влетает с края.
  - Напрраво: появляется на x=-5, летит вправо в экран
  - Налево: появляется на x=325, летит влево в экран

### Изменено
- **Версия:** поднята до 1.0.117 (index.html, main.js, base.js).

## [1.0.116] — 2026-06-27

## [1.0.112] — 2026-06-27

### Добавлено
- Финишная зона (goal) на Temple.
- Статический `Set collectedCoins` — монеты не появляются при возврате на уровень.
- Метод `createCoins()` в base.js.
- `checkFall()` — респавн при падении за экран (y > 280).

### Исправлено
- `staticSprite` + `group.add()` → `group.create()` (двойное добавление монет).
- `win()` передаёт `coinCount` при переходе.
- `win()` не падает при `nextScene === null` (конец игры).
- Touch-флаги сбрасываются в `init()`.
- Порядок создания: player → goal → coins (был баг с undefined).
- Версия вынесена в константу.

### Удалено
- `checkers.html` — шашки больше не в проекте.
